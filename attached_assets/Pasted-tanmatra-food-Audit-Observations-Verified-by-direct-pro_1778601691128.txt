P0 issues (fix immediately)
1. Marketing claim ↔ data mismatch (credibility risk)
Homepage states "24 RD Advisors" with prominence
/team page lists 3 RDs (Dr. Anjali Nair, Dr. Vikram Sethi, Dr. Kavya Menon)
/clinical shows "2 CLINICAL RDs" stat
/wellness heading: "1 RDs for wellness coaching" (also a singular/plural grammar bug)
Either staff up to 24 or change the home stat to match reality (~3). First reporter who counts will publish the discrepancy.

2. Broken nav labels → 404 destinations
The persistent top nav shows Eat / Plan / Orders / Community / Account — but these labels don't match the actual route names. So /eat, /plan, /community all 404 even though equivalent pages exist as /menu, /plans, /team. The nav anchors must be repointed to existing routes.

3. robots.txt and sitemap.xml return SPA HTML
GET /robots.txt    → returns full HTML page (200 OK, content-type: text/html)
GET /sitemap.xml   → returns full HTML page (200 OK, content-type: text/html)

Search engines cannot index this site. For a content-heavy nutrition site in India competing on therapeutic-meal queries, this is significant SEO loss.

4. Footer links → 404
/health-assessment, /clinical-menu, /therapeutic-plans, /for-dietitians, /assessment (homepage CTA) — none exist. Either repoint to /preferences, /clinical, /plans, /rd-partners respectively, or build them.

5. Unverified clinical claims (regulatory exposure)
"HbA1c reduction of 1.2% observed in diabetic patients after 12-week MNT adherence"
"Demonstrated 23% improvement in time-to-exhaustion after 6-week adherence"
Under India's ASCI Code and the Drugs and Magic Remedies (Objectionable Advertisements) Act, unsubstantiated therapeutic claims are actionable. Either publish the underlying study/citation or remove the specific numbers.

6. Missing regulatory disclosures
No FSSAI license number displayed (only "FSSAI Licensed" text). FSSAI requires the 14-digit license number on packaging and on the website.
No DPDP Act 2023 consent banner for personal/health data collection
No privacy policy or terms of service linked anywhere visible
P1 issues
7. Cloud Run cold-start UX
Backend is wellness-foods-1076775857511.asia-south2.run.app (Delhi region). Multiple pages show prolonged "Loading..." states (/wellness, /preferences, /rewards, /subscriptions, /recipes, /marketplace, /meal-planner, /challenges). Consistent with Cloud Run cold starts on min-instances=0. Either set min-instances ≥ 1, or add proper skeleton states with timeout/retry instead of indefinite "Loading…".

8. Single 1MB JS bundle, no code splitting
/assets/index-BRqVIZZL.js = 1,073KB uncompressed. For a 60-route SPA, this is a meaningful TTI/LCP penalty on Indian mobile networks. Should be route-split (Vite supports this with React.lazy + dynamic import()).

9. No /search page despite Cmd+K being prominent
The header shows "Search ⌘K" prominently. /search 404s. The Cmd+K modal works on the menu page only. Either build a global /search route or remove the keyboard hint from the header on pages where it doesn't work.

10. /signup 404
Login flow is OTP (no separate signup), which is fine — but the route /signup is broken. Redirect /signup → /login to handle inbound traffic from anywhere referencing it.

11. Inconsistent auth-gated UX
/appointments, /checkout-appointment render completely blank without auth. Other pages (/rd-console) correctly redirect to /login?next=.... Standardize on the redirect pattern.

12. Concept duplication in nav
/plans, /meal-planner, /subscribe, /subscriptions are four similar concepts. /team and /community (404) — what's the intended difference? User confusion likely. Worth a content audit.
