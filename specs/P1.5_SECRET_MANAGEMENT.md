# [P1.5] Secret Management — secrets off env defaults in prod

> Grounded in `brand-digital-twin` @ `fb03ddd`. `config.ts` reads everything from
> `process.env` **with mock fallbacks baked in** (`jwtSecret` →
> `'default-super-secret-key-9988'`, `masterKey` → `Buffer.alloc(32,'a')`,
> `supabase.key` → `'mock-secret-key-12345'`). `validateEnv()` (`config.ts:64`,
> invoked at `:98`) already refuses to boot if mock creds are present outside
> `NODE_ENV=test` — that guard is correct and stays. The gap is *where prod secrets
> come from*: env files, not a manager.

## Build
1. **`SecretProvider` interface** — `get(key): Promise<string>`, two impls:
   - `EnvSecretProvider` (local/test — reads `process.env`, today's behavior).
   - `ManagedSecretProvider` (prod — vault/KMS/Secret Manager; cached with TTL).
   Config selects by `NODE_ENV` / a `SECRET_PROVIDER` flag.
2. **Route the sensitive bootstrap secrets through it**: `jwtSecret`, `masterKey`
   (vault AES key), `supabase.key`, plus OAuth client secrets and the developer
   token. The mock fallbacks remain reachable **only** when `NODE_ENV=test`.
3. **`validateEnv()` unchanged** as the boot guard — it already encodes the
   "no mock creds in prod" invariant; ensure it runs *after* the provider resolves
   so a manager-supplied secret counts as present.
4. **Rotation-friendly**: `ManagedSecretProvider` re-reads on TTL expiry so a
   rotated key is picked up without redeploy. (The credential vault's existing
   AES-256-GCM stays as-is for *stored* tokens — this is about *bootstrap* secrets.)

## Checklist
- [ ] `SecretProvider` interface + env/managed impls
- [ ] The bootstrap secrets resolve via the provider; mock defaults gated to test
- [ ] `validateEnv()` runs post-resolution; still rejects mocks outside test
- [ ] Never log a resolved secret; redaction (P1.2a scrubber) covers them
- [ ] Tests: prod-mode boot with managed provider succeeds; prod-mode boot with a
      mock default → refuses

## Done when
Prod boots with zero secrets in env files and `validateEnv` still rejects mock creds.
